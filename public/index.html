What is Node.js?

Node.js is an open-source, cross-platform runtime environment that allows developers to create all kinds of server-side tools and applications using JavaScript outside of the browser. (i.e. running directly on a computer or server OS).
More info. Node.js

Benefits of using NodeJs from web server development perspective

  • Great performance! Node was designed to optimize throughput and scalability in web applications and is a good solution for many common web-development problems.

  • Code is written in "plain old JavaScript", which means taht less time is spent dealing with "context shift" between languages when you are writing both, client-side and server-side. (FrontEnd-BackEnd)

  • Many other new and popular languages compile/convert into JavSScript so you can also use TypeScript, CoffeeScript, ClojureScript, Scala, LiveScript, etc.

  • The node package manager (NPM) provides access to hundreds of thousands of reusable packages, It also has best-in-class dependency resolution and can also be used to automate most of the build toolchain.

  • NodeJs is portable. Is is available on Microsoft Windows, macOS, Linux, Solaris, FreeBSD, OpenBSD, WebOS, and NonStop OS. Is is well supported by manu web hosting providers, that often provide specific infrastructure and documentation for hosting Node sites.

  • Is has a very active third party ecosystem and developer community, with lots of people who are willing to help.

What is Express.js?

Express is a minimal and flexible and most popular NodeJs web application framework that provides a robust set of features for web and mobile applications. More info -> Expressjs.com

It provides mechanisms to:

  • Wirte handlers for requests with different HTTP verbs at different URL paths (routes).

  • Integrate with "view" rendering engines in order to generate responses by inserting data into templates.

  • Set common web applications settings like the port to use for connecting, and the location of templates that are used for rendering the response.

  • Add additional request processing "middleware" at any point within the request handling pipeline.

While Express itself is fairly minimalist, developers have created compatible middleware packages to address almost any web development problem. There are libraries to work with cookies, sessions, users logins, URL parameters, POST data, security headers, and many more.
You can find a list of middleware packages maintained by the Express team at Express Middleware

What is Middleware?

Middleware is software that lies between an operating system and the applications running on it. Essentially functioning as hidden translation layer, middleware enables communication and data management for distributed applications. It's sometimes called plumbing, as it connects two applications together so data and databases can be easily passed between the "pipe". Using middleware allows users to perform such requests as submitting forms on a web browser, or allowing the web server to return dynamic web pages based on a user's profile.

Common middleware examples include database middleware, application server middleware, message-oriented middleware, web middleware, and transaction-processing monitors. Each program typically provides messaging services so that different applications can communicate using messaging frameworks like simple object access protocol (SOAP), web services, representational state transfer (REST), and JavaScript object notation (JSON), While all middlware performs communication functions, the type a company chooses to use will depend on waht service is being used and what type of informtacion needs to be communicated. This can include security authentication, transaction management, message queues, applications servers, web servers, and directories. Middleware can also be used for distributed processing with actions occurring in real time rather tan sending data back and forth.

Functions that execute during the request to the server, each middleware function has access to request and response object. It is between req => middleware => res . It is used to avoid boilerplate code, just by creating a middleware function, the next step is calling that function in the app.(method).
